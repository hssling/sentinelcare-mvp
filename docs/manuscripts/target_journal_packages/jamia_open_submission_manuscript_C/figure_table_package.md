# Figure and Table Package

## Figures

### Figure 1. Governance as a runtime function

Core runtime questions:

- what logic is active?
- who approved it?
- what evidence supported approval?
- what alerts and overrides occurred under it?
- can it be rolled back immediately?

### Figure 2. Policy lifecycle model

States:

- draft
- pending approval
- approved active
- approved inactive or superseded
- retired

### Figure 3. Four-stage validation pathway

Stages:

1. retrospective benchmarking
2. silent prospective validation
3. controlled live pilot
4. conditional scale-up

## Tables

### Table 1. Minimum deployable safety controls

| Control | Function |
|---|---|
| Evidence-bearing alerts | reduce opaque signaling |
| Source transparency | support review and trust |
| Owner plus deadline | ensure accountable action |
| Queue and escalation state | track unresolved work |
| Override capture | preserve dissent and learning |
| Versioned activation | support change control |
| Rollback capability | enable rapid containment |
| Validation artifacts | support release decisions |

### Table 2. Policy lifecycle states

| State | Meaning |
|---|---|
| Draft | under development |
| Pending approval | awaiting governance decision |
| Approved active | live in workflow |
| Approved inactive or superseded | retained, not live |
| Retired | removed from future use |

### Table 3. Validation stages and aims

| Stage | Aim |
|---|---|
| Retrospective benchmarking | estimate fidelity and obvious failure modes |
| Silent prospective mode | quantify burden and timing without intervention |
| Controlled live pilot | test bounded live use with oversight |
| Conditional scale-up | expand only after stability and governance maturity |

### Table 4. Governance metrics

| Metric | Governance purpose |
|---|---|
| Harmful false-positive rate | detect unsafe signaling |
| No-action-after-alert rate | detect low-value alerts |
| Override rate and rationale mix | assess trust and burden |
| Closure completeness | assess end-to-end accountability |
| Escalation latency | assess response reliability |
| Alert burden by role | detect inequitable workflow load |
| Change impact after updates | assess release safety |
